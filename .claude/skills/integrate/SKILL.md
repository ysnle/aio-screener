# 자동 자료 통합 스킬 (v2)

## 트리거 조건
사용자가 아래 유형의 자료를 보내면 자동 실행:
- 분석글/칼럼 (매크로, 반도체, 시장 분석)
- IB 리서치 노트 (JP모건, 골드만, Wells Fargo, KeyBanc 등)
- 인터뷰 내용 (젠슨 황, 파월 등)
- 시장 데이터/가격 동향 (DRAMeXchange, TrendForce 등)

---

## 핵심 원칙

> 등급/목표주가는 결과물이다. 반영해야 할 것은 그 결론을 만든 **사고 구조**와 **논리 체계**다.
>
> "NVDA OW PT $175" — 이것은 반영하지 않는다.
> "AI 이질화(heterogenization)가 진행될수록 MRVL은 구조적 필수 인프라로 진화한다" — 이것을 반영한다.

---

## 실행 워크플로우

### 1단계: 자료 유형 분류 + 핵심 티커 목록 추출

자료 종류 판별:
- IB 리서치 노트 → 투자 테시스 + 데이터 추출
- 매크로/시장 분석 → 시그널 + 프레임워크 추출
- 인터뷰/발언 → 관점 + 포지셔닝 추출
- 다중 리포트 (같은 이벤트를 다른 하우스가 다룬 경우) → 관점 수렴/분기 추가 추출

원문에서 언급된 **핵심 티커** 목록을 먼저 뽑는다. 나중에 E2 eval에서 사용.

---

### 2단계: 프레임워크 추출

두 레이어를 순서대로 추출한다. 레이어 1(데이터)을 끝낸 뒤 레이어 2(프레임워크)를 반드시 추출한다.

#### 레이어 1 — 데이터 추출
- 핵심 수치 (숫자, 비율, 날짜)
- 인과 관계 체인 (A → B → C 명시)
- 수혜/피해 종목·섹터
- 시나리오 분기 (조건부 분기점 포함)
- 시그널 지표 + 임계값

#### 레이어 2 — 프레임워크 추출 (핵심)

아래 5개 질문을 원문에 대고 명시적으로 답한다. 답이 없으면 "해당 없음"으로 표시.

**Q1. 테시스 로직**: 이 애널리스트가 컨센서스와 *다르게* 보는 이유가 무엇인가?
- 단순한 수치 차이가 아니라 *왜* 다르게 보는가를 한 문장으로.
- 예: "WF가 AMD를 보는 이유: GPU가 NVDA를 이길 필요 없다 — '충분히 좋은 + 비용 우위 + 약정 확인' 3중 탈리스크 구조."

**Q2. 패러다임 전환**: 기존 분석 틀을 깨는 새 프레임워크인가?
- 있으면: 기존 틀 vs 새 틀을 대비 형식으로 명시.
- 예: "기존: Capex↑ → 공급↑ → 가격↓. 새: HBM이 DRAM 팹 용량 3~4x 잠식 → Capex↑에도 공급↑ 불가."

**Q3. 핵심 논쟁 변수**: 이 테시스가 틀리려면 무엇이 바뀌어야 하는가?
- Bull 가정 vs Bear 가정을 각각 한 줄씩.
- 예: "Bull: Diamond Rapids 경쟁력 미복구 + ARM 침식 느림. Bear: Intel 반격 성공 or ARM 서버 가속."

**Q4. 구조적 논리**: 표면 이벤트 뒤에 있는 역학 구조는 무엇인가?
- 발표/이벤트(LTA 체결, 파트너십 발표 등)를 만든 *역학*을 추출.
- 예: "AVGO LTA 뒤 역학: TPU 단독 ≠ 경쟁력, TPU + AVGO 네트워킹 = 경쟁력 → 네트워킹이 시스템 성능 결정자 = 협상 레버리지."

**Q5. 인접 파급**: 이 뷰가 유효하면 어떤 인접 종목/섹터의 thesis가 연동해서 바뀌는가?
- 직접 언급된 종목 외에 논리적으로 연결되는 종목·섹터를 추론.
- 예: "WFE 구조적 성장 테시스 → LRCX/AMAT를 사이클주 밸류에이션이 아닌 성장주로 재분류해야 함."

**다중 리포트 처리 (같은 이벤트를 여러 하우스가 다룰 때 추가):**
- 관점이 수렴하는 지점: 여러 하우스가 동의하는 것 (컨빅션 강함)
- 관점이 갈리는 지점: 하우스별로 다르게 보는 것 (불확실성 높음, 반드시 표시)
- 예: "MRVL-NVDA 파트너십 — 수렴: ALAB이 가장 임팩트 큰 수혜주(MS/Evercore 동의). 분기: MS는 커스텀 실리콘에 회의적(CY27 팩터 아님), JPM은 즉시 수혜."

### 3단계: 스크리너 반영

레이어 1 → SCREENER_DB 메모 + TECH_KW/MACRO_KW 키워드
레이어 2 → CHAT_CONTEXTS (프레임워크 레이어만 들어가는 곳)

#### SCREENER_DB 메모 작성 기준
- 레이어 1 데이터를 압축해서 반영
- 형식: `[출처] 핵심 포인트 · 핵심 포인트 · ...`
- 목표주가는 `PT $X(하우스)` 형식으로만, 테시스 없이 단독 기재 금지

#### TECH_KW / MACRO_KW 추가 기준
- 새 개념/현상/제품명이 뉴스에 등장할 때 해당 용어 추가
- 3글자 미만 단독 키워드 금지 (R17)
- 추가 후 비금융 일반 텍스트에서 오탐 여부 머릿속에서 체크

#### CHAT_CONTEXTS 작성 형식

```
[{제목} — {출처} {날짜}] {레이어 2 프레임워크 중심으로 서술}.
{Q2 패러다임 전환이 있으면}: 기존 틀 vs 새 틀을 명시.
{Q3 핵심 논쟁}: Bull 가정 / Bear 가정.
{Q4 구조적 논리}: 이벤트가 아닌 역학.
{Q5 인접 파급}: 연동 종목/섹터.
```

**나쁜 예 (데이터만 나열):**
```
[MRVL — MS 04/01] NVDA $2B 투자, NVLink Fusion, PT $103.
```

**좋은 예 (프레임워크 중심):**
```
[MRVL 이진 선택 해소 테시스 — JPM 04/01] MRVL 없으면 하이퍼스케일러는 양자택일:
"NVDA 순정" vs "완전 자체 제작." MRVL NVLink Fusion IP블록 = 이진 선택을 스펙트럼으로
전환. 인접 파급: AI 이질화 심화 → MRVL 구조적 필수 인프라 진화. ALAB:
NVSwitch 대체 = 스케일업 네트워킹 그린필드 선두.
```

#### 반영 우선순위 (모든 걸 다 담을 수 없을 때)
1. Q2 패러다임 전환 (기존 틀을 깨는 것이면 최우선)
2. Q4 구조적 논리 (이벤트 뒤의 역학)
3. Q3 핵심 논쟁 변수 (Bull/Bear 분기점)
4. Q5 인접 파급
5. Q1 테시스 로직
6. 레이어 1 데이터

### 4단계: 버전 범프 + CHANGELOG

- 버전 6곳 동기화 (R1): title, badge, APP_VERSION, version.json, _context/CLAUDE.md, CHANGELOG.md
- CHANGELOG.md 항목: 통합 내용 요약 (종목 수, 키워드 수, 주요 프레임워크)

### 5단계: 자기평가 (Binary Eval Check)

통합 완료 전 아래 6개 항목에 **명시적으로 yes/no** 답변. no가 하나라도 있으면 해당 항목 수정 후 재통합.

| # | 평가 항목 | 기준 |
|---|-----------|------|
| E1 | 키워드 추가 | TECH_KW 또는 MACRO_KW에 ≥3개 신규 키워드 |
| E2 | SCREENER_DB 갱신 | 원문 핵심 티커 중 ≥1개 메모 업데이트 |
| E3 | 프레임워크 레이어 반영 | Q1~Q5 중 ≥2개 답변이 CHAT_CONTEXTS에 명시적으로 반영됨 |
| E4 | CHAT_CONTEXTS 형식 | 데이터 나열이 아닌 논리/역학 중심으로 서술됨 |
| E5 | CHANGELOG 기록 | CHANGELOG.md 최상단에 이번 버전 항목 존재 |
| E6 | R17 준수 | 추가된 모든 키워드 ≥3글자 |
| E9 | 복리 환류 | Q2(패러다임 전환) 또는 Q4(구조적 논리) 발견 시 KNOWLEDGE-BASE.md에도 인사이트 축적했는가? |

> E3 no → Q1~Q5 답변 중 가장 강한 프레임워크 2개 이상을 CHAT_CONTEXTS에 추가.
> E4 no → CHAT_CONTEXTS 해당 항목을 "좋은 예" 형식으로 재작성.
> 나머지는 해당 항목 보완 후 재체크.

## Gotchas (과거 통합 작업에서 발생한 실수)

1. **등급/목표주가만 반영** — "OW PT $345" 추가로 작업 완료 착각. 이것은 레이어 1의 데이터도 아닌 단순 결과물. 반드시 레이어 2까지 추출해야 함.

2. **CHAT_CONTEXTS 데이터 나열** — `[AMD — WF] PT $345, EPS $6.25, EPYC +40%` 식의 나열은 금지. 논리 구조와 역학 중심으로 서술.

3. **다중 리포트에서 관점 혼합** — 여러 하우스가 다른 뷰를 가질 때 하나로 합쳐서 쓰면 정보 손실. 수렴/분기 지점을 명시.

4. **CHAT_CONTEXT 이원화 미적용** — 주가/지수는 `_closeSnap()` 종가 기준, VIX/DXY 등 시장환경은 `_liveSnap()` 실시간. R13 준수.

5. **키워드 3글자 미만 추가** — TECH_KW에 `'S'`, `'AI'` 같은 단일/이중 문자 → 모든 텍스트에서 오탐. R17 준수.

6. **매크로 뉴스에 ETF 티커 노출** — 지정학/정책 뉴스에 SPY, QQQ 티커 표시 → R16 준수.

7. **레퍼런스 파일 경로 오류** — 레퍼런스 .md 파일 존재 확인 후 병합.

8. **Q2 패러다임 전환을 놓침** — 가장 중요한 프레임워크인데 가장 자주 빠짐. 기존 분석 틀과 새 프레임워크를 대비 형식으로 명시해야 함.

## 뉴스/소식 자료 처리 (v46.1 신설)

### 뉴스가 IB 리서치와 함께 올 때

사용자가 뉴스/소식(마켓 서머리, 기업 뉴스, CEO 서한 등)과 IB 리서치를 함께 보내면:

1. **파이프라인 교차 분석**: 뉴스의 시장 맥락(유가 변동, 섹터 흐름, 자금 흐름)과 IB 리서치의 기업 분석을 교차하여 "왜 이 뉴스가 이 종목에 영향을 미치는지" 역학 추출.
   - 예: "WTI -16% 급락(뉴스) + Anthropic ARR $30B(BofA) = 에너지→기술 섹터 로테이션 동력"

2. **뉴스 수집 파이프라인 점검**: 보내준 뉴스 속 이벤트/종목/키워드가 현재 스크리너의 수집 로직(TECH_KW/MACRO_KW/NEWS_SOURCES/KNOWN_TICKERS)에서 **잡히는지** 확인.
   ```bash
   # 뉴스 속 키워드가 TECH_KW/MACRO_KW에 있는지
   grep -n "'{키워드}'" index.html | grep "TECH_KW\|MACRO_KW"
   # 뉴스 속 종목이 KNOWN_TICKERS에 있는지
   grep "'{TICKER}'" index.html | grep "KNOWN_TICKERS"
   # 뉴스 소스가 AIO_NEWS_SOURCES에 등록되어 있는지
   grep "'{소스명}'" index.html | grep "NEWS_SOURCES\|_newsSources"
   ```

3. **누락 시 즉시 추가**:
   - 키워드 누락 → TECH_KW/MACRO_KW 추가 (R17 3글자+ 준수)
   - 티커 누락 → KNOWN_TICKERS + SCREENER_DB 동시 추가
   - 소스 누락 → 해당 소스의 RSS/API 가용성 확인 후 추가 가능 여부 리포트

### 뉴스만 올 때 (IB 리서치 없이)

1. 뉴스 내용에서 시장 영향 종목/섹터 추출
2. 해당 종목 SCREENER_DB 메모 갱신
3. HOME_WEEKLY_NEWS 교체 후보 검토 (score 90+ 이벤트면)
4. CHAT_CONTEXTS 시나리오 수치가 뉴스와 괴리되면 갱신
5. 수집 파이프라인 점검 (위 2번 동일)

### Self-Eval 추가 항목

| # | 항목 | 기준 |
|---|------|------|
| **E7** | 뉴스 파이프라인 점검 | 보내준 뉴스 속 핵심 키워드/티커가 수집 로직에 등록되어 있는지 확인했는가? |
| **E8** | 교차 분석 | 뉴스의 시장 맥락과 IB 리서치를 교차하여 역학을 추출했는가? (뉴스 단독 나열 아님) |

## v46.4 추가: 페이지별 반영 체크리스트

> **핵심 원칙**: CHAT_CONTEXTS에만 반영하면 안 된다. 자료의 프레임워크/데이터가 영향을 미치는 **모든 페이지의 정적 텍스트·시스템 프롬프트·데이터 파이프라인**에 반영해야 한다.

### 반영 대상 체크리스트 (통합 완료 후 순회)

```
[ ] CHAT_CONTEXTS §N 추가 (프레임워크 레이어)
[ ] SCREENER_DB 해당 티커 메모 갱신
[ ] TECH_KW / MACRO_KW 신규 키워드 추가
[ ] macro system prompt: Fed 정책 경로 확률, 유가 래더 등 갱신 필요?
[ ] fxbond system prompt: 금리/채권/TGA 관련 맥락 갱신 필요?
[ ] fundamental system prompt: 섹터 로테이션, 사이클 가이드 갱신 필요?
[ ] themes system prompt: Capex 효율화 계층, DC전력 계층 갱신 필요?
[ ] signal system prompt: 바닥 프로세스, 스코어 기준 갱신 필요?
[ ] sentiment system prompt: 극단값 기준, HY OAS 갱신 필요?
[ ] technical system prompt: 매크로-기술적 교차 분석 갱신 필요?
[ ] 시그널 CP 카드 (CP1~CP8): 해당 카드 detail 텍스트 갱신 필요?
[ ] 매크로 시나리오 트리: 시나리오 조건/확률 갱신 필요?
[ ] 데일리 브리핑 캘린더: 이벤트 추가/수정 필요?
[ ] HOME_WEEKLY_NEWS: score 90+ 이벤트면 교체 검토
[ ] 한국 거시 페이지: 금리차/수급/포지셔닝 텍스트 갱신 필요?
[ ] KR_THEME_CATALYSTS: 한국 테마 촉매 텍스트 갱신 필요?
[ ] _generateAIBriefing 프롬프트: 매크로 맥락 블록 갱신 필요?
[ ] _sectorRRGSeed / _SECTOR_PCT_FALLBACK: 섹터 로테이션 이벤트면 갱신
[ ] KNOWLEDGE-BASE: Q2/Q4 패러다임 전환 발견 시 인사이트 축적
```

### 반영 대상 판단 기준
